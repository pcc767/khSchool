JQuery의 개요

- 경량 JavaScript 라이브러리로 복잡한 JS코드를 단축표현 및 반복문법을 제공함

- 개발자가 편리하고 생산성 있도록 만들어주는 라이브러리이다.

JQuery의 장점

1) DOM 관련 처리를 쉽게 할수 있음 -> 생산성, 편리성 증대

2) Ajax 통신 또는 UI 이벤트를 쉽게 구현 가능 하다.

3) 별도의 플러그인 없이 차트, 슬라이드쇼, 테이블들을 구현 가능 (하나 난 쓸래..)

JQuery의 단점

1) 웹표준이 아니다. 2) 숙련도가 필요하다. 3) 웹 프론트의 복잡성 증가

JQuery 로드 방법

1. 파일로 로드하는 방법 (공식 홈페이지에서 다운로드 가능)

2. CDN (Contents Delivery Network)를 통한 인터넷 상에서 로드하는 방법

3. CDN + 특정버전 + 보안기능

#(docmemt).ready()

- html 페이지를 모두 로드하고 나서 ready 메소드를 실행하는 개념으로 document.onload 이벤트와 동일

- 왜 사용하느냐? : html script를 load할 경우 태그가 렌더링 되기 전에 호출되면 js가 동작하지 않기 때문에

- ready 함수 호출을 통해 html렌더링 이후 모두 로드 된 상태에서 js코드를 호출하기 위해 사용한다.
